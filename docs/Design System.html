Ein datengetriebenes, dunkles Operator-Interface für Stadtklima-Analyse und Resilienz-Simulation. Dieses Dokument definiert Tokens, Komponenten, Layouts und Konventionen — als verlässliche Referenz für Engineering und Claude Code.

Designprinzipien

Fünf Leitsätze, die jede Designentscheidung durchziehen.

01 / Daten zuerst

Lass Daten atmen.

Zahlen tabular, monospaced, mit Einheit. Keine dekorativen Charts. Keine Stockbilder. Jeder Wert verdient seinen Platz.

02 / Farbe ist Signal

Akzentfarben tragen Bedeutung.

Grün = Klimagewinn, Amber = Hitze, Rot = kritisch, Blau = Wasser/Info, Lila = Vulnerabilität. Niemals dekorativ einsetzen.

03 / Karte ist Anker

Räumliche Daten brauchen Raum.

Die Karte dominiert die Seite (≥ 60 % Breite). Steuerung sitzt rechts, Interpretation darunter. Nie Karte als Beilage.

04 / Dichte mit Atem

Operator-Tool, nicht Marketing.

Hohe Informationsdichte ist OK — solange Hierarchie klar ist. 8 px Grid, klare Gruppen, sparsame Trenner.

05 / Spekulation kennzeichnen

Modell ist nicht Realität.

Simulationsergebnisse, Auto-Interpretationen und Prognosen werden immer als solche markiert (Chip "auto-generiert", Konfidenz, Modellname). Verwechslungsgefahr mit gemessenen Daten ist die schlimmste UX-Sünde.

function KpiCard({ label, value, unit, sub, color, trend, sparkline, icon }) {
  const c = COLORS[color]; // { fg, bg, border }
  return (
    <div className="card p-5">
      <div className="flex items-start justify-between">
        <div>
          <Label>{label}</Label>
          <Value color={c.fg}>{value}<Unit>{unit}</Unit></Value>
          <Sub>{sub}</Sub>
        </div>
        <IconTile color={c} icon={icon} />
      </div>
      {sparkline && <Sparkline data={sparkline} />}
      {trend && <Trend {...trend} />}
    </div>
  );
}

  • · Unter der Karte: Mini-Bilanz (Fläche, Kosten, Förderung) in 4er-Grid.

Accessibility

Operator-Tool — desktop-only — aber Basis-A11y bleibt Pflicht.

  • · Kontrast: text-0 auf bg-0 = 14:1 ✓ · text-2 auf bg-1 = 4.5:1 ✓ · text-3 nur für Meta, nie für Aktions-Labels.
  • · Farbe nie alleinsprechend: Heat-Bars zeigen Wert + Farbe; Score-Cells haben Bar + Zahl.
  • · Focus-Ring: 2 px var(--green) outline + 2 px offset auf allen interaktiven Elementen.
  • · Slider: keyboard ↑/↓/←/→ in step-Inkrementen.
  • · Toggle: role="switch", aria-checked.
  • · Sprache: html lang="de" — wichtig für Screen-Reader-Aussprache deutscher Begriffe.
  • · Bewegung: prefers-reduced-motion respektieren — pulse-dot und spinner werden zu statischen Indikatoren.

Format-Utils

// utils/format.ts
export const fmt = {
  num:  (v: number, d = 0) => v.toLocaleString("de-DE", { maximumFractionDigits: d, minimumFractionDigits: d }),
  temp: (v: number) => `${v.toFixed(1)}°C`,
  dT:   (v: number) => `${v > 0 ? '+' : ''}${v.toFixed(2)} °C`,
  pct:  (v: number, d = 1) => `${v.toFixed(d)}%`,
  area: (m2: number) => m2 >= 10000
    ? `${(m2/10000).toFixed(2)} ha`
    : `${m2.toLocaleString("de-DE")} m²`,
  index: (v: number) => v.toFixed(2),
};

System-Prompt-Snippet
You are working on "Resilientes Würzburg", a dark, data-driven climate
resilience dashboard for the city of Würzburg.

UI rules — non-negotiable:
- Background #0f1117, surface #151823, elevated #1a1e2c.
- Inter for UI, JetBrains Mono for ALL numbers (with tabular-nums).
- Semantic colors carry meaning, never decoration:
  green=climate gain, amber=heat, red=critical, blue=water, purple=vulnerability.
- 8px grid. Card radius 10px. Button/input radius 6px.
- Numbers in de-DE format (1.200, 38,4). Temperature with °C, no space.
- All AI/model output gets a "auto-generiert" chip and a confidence value.
- No marketing language. German UI, English only for established
  technical terms (LST, HVI, API, Layer, Dashboard).

Layout:
- 220px fixed left sidebar (bg-1).
- Sticky topbar with breadcrumb.
- Two-column page layout: 1fr map / 280–400px right rail.
- Section gap 48px, card gap 16px.

Components are documented in "Design System.html". Always check there
before inventing new components. Reuse: KpiCard, OutputCard, Toggle, Slider,
Dropdown, MapSurface, InterpretationBox.

Motion: 150ms ease-out for hover, 300ms for layer reveals, 500ms for sim
results. No bounce, no spring, no entrance animations on data.

Accessibility: focus rings green, color never sole carrier of meaning,
prefers-reduced-motion respected.
Standard-Workflow für neue Features
  1. 1. Komponente schon vorhanden? → in components/ prüfen, dann hier im DS suchen.
  2. 2. Neue Daten-Visualisierung? → erst die Format-Utils erweitern, dann das Chart in charts/.
  3. 3. Neuer Layer auf der Karte? → map/overlays/ + LayerPanel-Eintrag + Farbsemantik klären.
  4. 4. Neue Sim-Seite? → Pattern "Simulation Layout" verwenden, useSimulation-Hook abstrahieren.
  5. 5. Neuer KPI? → erst Color-Map prüfen, dann Trend-Logik (siehe Voice §Trends).

Don'ts
  • · Keine neue Akzentfarbe einführen ohne Designer-Review.
  • · Keine Inter Black, kein dritter Font.
  • · Keine Bounce-Animations, keine Page-Transitions.
  • · Keine Emojis im UI (außer Brand-Logo, falls definiert).
  • · Keine Stockbilder, keine generierten Hero-Illustrationen.
  • · Keine ungekennzeichneten Modell-/LLM-Outputs.
  • · Keine englischen UI-Strings ohne Pflicht-Grund.
  • · Keine Charts mit Animation beim Mount.
  • · Keine echten Karten-Tiles ohne Lizenz-Klärung.
